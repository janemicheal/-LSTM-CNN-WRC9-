# WRC9自动驾驶系统

## 项目概述

WRC9自动驾驶系统是一个基于深度学习的游戏自动驾驶解决方案，能够实时捕获游戏画面，通过训练好的模型预测最优操作指令，并将指令转换为游戏输入，实现自动驾驶。

## 模块功能说明

### 1. 主程序 (main.py)
- 系统入口，负责初始化和协调各个模块
- 提供键盘监控功能，支持K键启动/停止自动驾驶
- 管理系统运行状态和线程

### 2. 屏幕捕获模块 (screen_capture.py)
- **核心职责**：实时捕获游戏画面
- **功能**：
  - 支持多显示器捕获
  - 可配置捕获区域和目标分辨率
  - 提供单帧捕获和多线程捕获两种模式
  - 支持ROI提取和画面增强
- **数据输出**：处理后的图像帧 (960×540 RGB)

### 3. 模型推理模块 (model_inference.py)
- **核心职责**：加载训练好的模型，预测最优操作指令
- **功能**：
  - 支持CUDA/CPU运行
  - 提供模型加载和优化功能
  - 支持输出平滑和灵敏度调整
  - 支持路书特征融合
- **数据输入**：图像帧 (960×540 RGB) + 路书特征
- **数据输出**：控制指令 (转向、油门、刹车)

### 4. 输入模拟模块 (input_simulator.py)
- **核心职责**：将控制指令转换为游戏输入
- **功能**：
  - 支持虚拟游戏手柄和键盘两种输入方式
  - 提供输入平滑和持续输入控制
  - 支持单次输入和按键模拟
- **数据输入**：控制指令 (转向、油门、刹车)
- **数据输出**：游戏输入事件

### 5. 路书识别模块 (roadbook_recognizer.py)
- **核心职责**：识别游戏中的路书信息
- **功能**：
  - 识别弯道方向和角度
  - 识别建议速度和路况
  - 提供路书特征向量
- **数据输入**：图像帧
- **数据输出**：路书特征字典

## 数据传递流程

```
游戏画面 → ScreenCapture → 图像帧 (960×540) → ModelInference
                                                          ↓
路书信息 → RoadbookRecognizer → 路书特征 → ModelInference → 控制指令 → InputSimulator → 游戏输入
```

## 使用方法

### 1. 环境准备

安装依赖：
```bash
pip install numpy opencv-python pillow mss torch vgamepad pynput keyboard pygetwindow
```

### 2. 启动系统

运行主程序：
```bash
python main.py --mode inference --debug
```

### 3. 操作说明

- **按K键**：启动/停止自动驾驶
- **按ESC键**：退出系统
- **按J键**：切换录制模式（对大小写不敏感，避免与游戏R键复位冲突）

### 4. 录制功能说明

#### 录制逻辑
1. **开始录制**：按J键开始录制，系统会初始化`SimpleRecorder`并开始录制
2. **实时记录**：录制过程中，系统会实时记录游戏画面、路书信息和操作指令
3. **停止录制**：再次按J键停止录制，系统会自动将录制的数据保存到指定目录
4. **数据保存**：录制的数据会保存到`data/raw/`目录下，文件格式为`recording_YYYYMMDD_HHMMSS.npz`

#### 录制状态
- **录制中**：状态显示窗口显示"录制状态: 📹 录制中"，指示灯为黄色
- **未录制**：状态显示窗口显示"录制状态: 📴 未录制"，指示灯为红色

#### 录制数据内容
- 游戏画面帧 (960×540 RGB)
- 路书识别文本
- 操作指令（转向、油门、刹车）
- 时间戳信息

#### 录制文件格式
录制的数据使用NPZ格式保存，包含以下字段：
- `frames`：游戏画面帧数组
- `timestamps`：时间戳数组
- `roadbook_infos`：路书信息数组
- `steerings`：转向指令数组
- `throttles`：油门指令数组
- `brakes`：刹车指令数组

### 5. 配置说明

系统支持通过配置文件或代码修改参数，主要配置项包括：

- **屏幕捕获**：
  - `target_resolution`：输出分辨率 (默认960×540)
  - `monitor_id`：显示器ID
  - `enable_roi`：是否启用ROI提取

- **模型推理**：
  - `model_path`：模型文件路径
  - `device`：运行设备 (cuda/cpu)
  - `enable_smoothing`：是否启用输出平滑

- **输入模拟**：
  - `input_method`：输入方式 (gamepad/keyboard)
  - `enable_smoothing`：是否启用输入平滑

## 训练说明

### 1. 数据录制

使用数据记录模块录制游戏数据：
```bash
python main.py --mode record
```

### 2. 数据预处理 

 对录制的数据进行预处理： 
 ```bash 
 python src/data_preprocessor.py 
 ``` 

 数据预处理模块会自动处理 `data/raw/` 目录下的所有录制数据，并将处理后的结果保存到 `data/processed/` 目录。

 ### 3. 模型训练 

 使用预处理后的数据训练模型： 
 ```bash 
 python src/model_trainer.py 
 ``` 

 模型训练模块会加载 `data/processed/` 目录下的预处理数据，并将训练好的模型保存到 `models/` 目录。

## 常见问题

### 1. 屏幕捕获分辨率不正确
- 检查显示器ID是否正确
- 确保`capture_region`设置为None（全屏捕获）
- 检查`target_resolution`是否设置为960×540

### 2. 自动驾驶无法启动
- 确保按K键切换自动驾驶状态
- 检查模型文件是否存在
- 检查输入设备是否正确初始化

### 3. 控制指令不准确
- 调整模型灵敏度参数
- 启用输出平滑功能
- 重新训练模型

## 注意事项

1. 确保游戏以窗口化或全屏模式运行在2560×1440分辨率
2. 首次运行前请检查模型文件是否存在
3. 使用虚拟游戏手柄需要安装vgamepad库
4. 建议在高性能GPU上运行以获得最佳效果
5. 自动驾驶状态下请勿手动操作游戏，以免干扰
